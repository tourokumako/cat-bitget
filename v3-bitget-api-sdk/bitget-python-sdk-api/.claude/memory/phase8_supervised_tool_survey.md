# phase8_supervised_tool_survey.md — 教師あり時系列分類 手法網羅リスト

作成日: 2026-04-27
位置づけ: 段階8（教師あり時系列分類）のツール選定一次資料。経緯ベースで網羅。

## 前提（必読・2026-04-27 改訂）

- **学習タスクの実効解像度は日単位1825サンプル（B案採用）**
  - 月単位60本/ハイブリッドは独立情報量60で頭打ち・L-129再発を構造的に防げないため却下
  - まこさんが日単位で5年×365日（=1825点）を手動ラベリング
  - 日次自己相関のため独立情報量は1825より少ない → ブロックCVで対処
- 段階方式: **直近1年365点を先行作成 → 5手法ベースライン → 進める/止める判定 → 合格時のみ残り4年に進む**
- ラベルクラス: UPTREND / DOWNTREND / RANGE（3クラス）
- 候補B 数値基準（参考閾値・最終評価軸）:
  - UPTREND 平均日次リターン > +1.22%
  - DOWNTREND 平均日次リターン < -0.82%
  - RANGE |平均日次リターン| < 0.34%
  - 状態継続中央値 ∈ [3, 30]日
- 評価本丸: **月単位の支配 regime が肉眼判定と一致するか**（L-126）

## 既知リスク（手法評価で常に参照）

- **L-129**: 教師あり完全フィットは 30-40% 天井（365日×50特徴・過学習）
- **L-134**: 異種特徴混合（ボラ/Funding/OI と方向系の混合）でモデルが支配される
- **L-132**: 同一カテゴリ複数特徴は冗長（事前に相関ヒートマップ確認）
- **L-126**: 月単位支配 regime が本丸・日内 regime 切替は不要
- **L-130**: 閾値はデータ駆動（mean / std / percentiles を必ず先に提示）
- **L-133**: WORKFLOW 既定タスク盲従禁止・上流前提を毎回疑う
- **L-127**: 過去却下方針への退路は禁止
- **L-131**: 各段階で 5-7 回調整不合格なら次段階へ・最大15-21回でストップ

## 2. 上位5手法 抽出（経緯と統計的根拠のみで選定）

選定基準:
- 60本の実効サンプル数で動く（c=○以上）
- 月単位形状認識に適合（d=○以上）
- L-129 試行済を除外
- 実装コスト e=○ または △（軽め）優先

| 順位 | 手法 | 選定根拠 | L-番号 / WORKFLOW根拠 |
|------|------|---------|---------------------|
| 1 | **MiniROCKET（C-2）** | 小サンプル時系列分類ベンチ上位・特徴偏り少（L-134対策）・実装軽量 | L-129 未試行・L-134 ランダムカーネルで支配回避 |
| 2 | **ROCKET（C-1）** | MiniROCKET の本家・実装容易・対照群 | 同上 |
| 3 | **1-NN + DTW（A-8）** | 時系列形状を直接比較・60本でも動作・解釈容易 | L-126 月単位形状一致と直結 |
| 4 | **TS2Vec（D-2）** | 自己教師あり事前学習で60本制約を緩和 | L-129 過学習対策の本命 |
| 5 | **Logistic Regression + 手作り集約特徴（A-1）** | 単純線形ベースライン・過学習リスク最小・解釈最容易 | L-130 データ駆動原則・他手法の比較ベース |

**意図的に除外した候補と理由**:
- LightGBM/XGBoost: A-2/A-3（RF/GB）が L-129 で天井確認済 → 同系列の追試は限界利益小
- LSTM/Transformer: 60本で学習不能（過学習確実）
- HIVE-COTE: 計算コストの割に MiniROCKET 比で限界利益小（公開ベンチ参照）
- InceptionTime: 60本では深層NN不適

## 3. 上位5手法 比較設計

### 評価軸（B案・日単位1825サンプル前提）

| 軸 | 指標 | 合格条件 | データ源 |
|----|------|---------|---------|
| 1 | 日単位合致率 | まこさん肉眼判定との一致率 ≥ 60% | ブロックCV（連続区間で5分割・時系列リーク防止） |
| 2 | 月単位合致率 | 予測ラベルの月内最頻値 vs まこさん月単位判定 ≥ 70% | 日予測 → 月モード集約 |
| 3 | UPTREND 数値 | 予測UPの平均日次リターン > +1.22% | 検証区間の日次集計 |
| 4 | DOWNTREND 数値 | 予測DTの平均日次リターン < -0.82% | 同上 |
| 5 | RANGE 数値 | 予測RGの \|平均日次リターン\| < 0.34% | 同上 |
| 6 | 継続性 | 予測ラベルの duration 中央値 ∈ [3,30]日 | 連続同一ラベルランの長さ |
| 7 | 過学習指標 | 学習合致率 − ブロックCV合致率 < 0.20 | 各手法の学習・検証スコア |
| 8 | 実装コスト | スクリプト書き上げ + 1回学習 < 半日 | 実装時間記録 |

### 手法別の追加チェック

| 手法 | 追加チェック |
|------|-------------|
| MiniROCKET | カーネル数（既定10000）の感度確認・60本では削減検討 |
| ROCKET | 同上・MiniROCKET との合致率差 |
| 1-NN+DTW | DTW窓制約（Sakoe-Chiba band）の感度・距離分布の妥当性 |
| TS2Vec | 事前学習用無ラベル区間の選定・fine-tune 過学習警戒 |
| LogReg | 集約特徴設計（月内 mean/std/return等）・特徴本数 ≤ 5 厳守（L-132） |

### 入力特徴設計（共通仕様・L-134遵守・B案対応）

- **方向系のみ**: 当日含む過去N日の MA20乖離・MA70乖離・連続上昇日数・ret_5d/10d/20d など
- ボラ系・Funding・OI は**入れない**（L-134・後処理で扱う）
- 各日の入力 = 当日までの過去30日（または60日）窓
- **未来情報禁止**（look-ahead 防止）: 当日のラベル予測には当日の close 含むが翌日以降は不可

## 4. 並行検証の手順案（1手法に賭けず複数同時走行）

### 全体方針

- 5手法を**同一データ・同一評価軸で並走**（L-127「退路提案禁止」・L-131「過剰最適化禁止」）
- 1手法あたり最大 5-7 回の調整（L-131）
- 5手法 × 平均3回 = 15ラウンド以内で**全手法のLOOCV合致率分布を確定**
- 不合格手法はその段階で打ち切り、合格候補のみ次フェーズ（実Replay検証）へ

### 段階構成（B案・段階方式）

| フェーズ | 内容 | 終了条件 |
|---------|------|---------|
| 8.0 | ラベル基準明文化（UP/DT/RG の定義文書） | `.claude/memory/phase8_label_criteria.md` 確定 |
| 8.1a | **直近1年365点ラベル先行作成（まこさん）** | `results/phase8_daily_labels_y1.csv` 確定 |
| 8.2a | 入力特徴設計・1年×日次特徴行列構築 | `results/phase8_features_daily_y1.csv` 確定 |
| 8.3a | **5手法ベースライン同時実装・ブロックCV 1巡** | 5手法すべての合致率・候補B数値・過学習指標が出揃う |
| **8.3.5** | **進める/止める判定ゲート（下記合格ライン）** | **GO → 8.1b / NOGO → 段階打ち切り** |
| 8.1b | 残り4年1460点ラベル作成（まこさん） | `results/phase8_daily_labels_y5.csv` 確定 |
| 8.2b | 5年×日次特徴行列再構築 | `results/phase8_features_daily_y5.csv` 確定 |
| 8.3b | 5手法ベースライン再実行（5年データ） | 合致率・数値・過学習指標を1年版と比較 |
| 8.4 | 上位2手法を選抜・各最大5回調整 | 合致率 ≥ 60% & 候補B数値合格 が出るか確定 |
| 8.5 | 合格手法の365d Replay 検証 | $/dt-day がベースライン超え or 失格確定 |
| 8.6 | 採用判定 / 段階2-5（HMM系）へ戻す判断 | まこさん最終判断 |

### 8.3.5 進める/止める判定ゲート（明文化・着手前確定）

**5手法のうち最低1手法**が以下を全て満たせば 8.1b へ進む。1手法も満たせなければ段階8 打ち切り。

| 軸 | 合格ライン | 不合格時の解釈 |
|----|-----------|--------------|
| 日単位合致率（ブロックCV） | **≥ 50%** | 365点ではここが上限と判断・5年に拡張する根拠が崩れる |
| 月単位合致率（日予測の月モード集約） | **≥ 65%** | L-126 月単位本丸を満たさない |
| UPTREND 数値 | 予測UPの平均日次リターン > **+1.0%** | 候補B（+1.22%）に近接していれば許容・データ少を考慮 |
| DOWNTREND 数値 | 予測DTの平均日次リターン < **-0.65%** | 同上（-0.82% に近接） |
| RANGE 数値 | 予測RGの \|平均日次リターン\| < **0.5%** | 同上（0.34% に近接・1年データの揺らぎ考慮） |
| 継続性 | 予測ラベルの duration 中央値 ∈ **[2, 45]日** | 5日刻み張り付き・即フリップを除外（段階6失敗の再発防止） |
| **過学習指標** | **学習合致率 − ブロックCV合致率 < 0.20** | **L-129 過学習再発確定（最重要ゲート）** |

**全手法不合格パターンの解釈と帰結**:
- 全手法が日単位合致率 < 40% → ラベル品質または特徴設計を疑う（8.0/8.2a に戻る）
- 全手法が過学習指標 ≥ 0.20 → 1年データでは構造的に不足（B案の前提崩壊・C案再検討）
- 上記以外で不合格 → 段階8 打ち切り・段階2-5 HMM系へ戻す

### 各フェーズで止まる条件（L-131・追加）

- 8.4 で合致率 60% に届かない → 段階8 を打ち切り段階2-5（HMM系教師なし）へ戻す
- 8.5 で実Replay が悪化 → look-ahead 剥離後の MA70 単純判定がベースラインを超えるかを再検証
