## 更新ルール（必読）

新規シグナル思いついたとき:
- §1 マスターリストに行追加（A〜H 既存カテゴリ or 新カテゴリ）
- §2 マッピングに行追加（◎/○/△/×/? を埋める）

実装したとき（scripts/signals/ に detect 追加）:
- §3 実装ステータス表を更新

検証実行したとき（analyze_signals.py 走らせた）:
- §5 検証ログに必ず1行追記
- §5 追記と**同時に** §2 マッピングのセルを更新:
  - NO-GO 判定 → そのレジーム×方向セルを `❌` に
  - 採用→Priority化 → セルを `✅` に
  - WARN → セルに `⚠` を併記して memo にログ参照を書く

Priority化（cat_v9_decider に組み込み）したとき:
- §1 既存対応列に P番号追加
- §2 マッピングのセルを `✅`
- §4 P番号→ID対応表を更新

セッション終了時の整合チェック（30秒）:
- §2 で `◎` または `○` が残っているセル × §5 検証ログと突合
- 検証済なのに `◎` のまま → `❌` または `✅` に直す

## 凡例

| 記号 | 意味 |
|------|------|
| ◎ | 本命候補（未検証） |
| ○ | 補助候補（未検証） |
| △ | 条件次第 |
| × | 不適（方向構造的に合わない・例: 順張りLONGをDOWNTRENDに使う等） |
| ? | 未評価（評価予定だが未着手） |
| ✅ | 検証済み有効（既存Priority稼働中・採用パラメータあり） |
| ❌ | 検証済み NO-GO（実績マイナス または REJECTED L-XXX） |
| ⚠ | WARN（要再検証・条件付き） |

## §5 採用判定基準

検証実行は scripts/analyze_signals.py を使用。L-118 対応として、干渉率・調整後NET・判定を必ず記録する。

- **GO**   : 調整後NET ≥ $3/regime-day かつ 干渉率 < 0.3
- **WARN** : 調整後NET ≥ $1/regime-day または 干渉率 0.3〜0.5（実装前に Replay 干渉テスト必須）
- **NO-GO**: 上記以外
