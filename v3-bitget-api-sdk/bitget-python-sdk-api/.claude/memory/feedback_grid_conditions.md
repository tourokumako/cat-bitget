グリッドサーチを走らせる前に、ベースライン（直前の比較グリッド）と条件が一致しているか必ず確認する。

**Why:** 2026-04-05、Priority別TP最適化グリッドを MAX_ADDS P23=1/P24=1（旧config値）のまま走らせてしまった。ベースラインの SHORT TP グリッドは P22/P23/P24 全て adds=5 で走らせていたため、条件不一致で結果が無意味になった。大量のトークンと時間を無駄にした。

**How to apply:** 新しいグリッドを提案する前に以下を確認する：
1. 前回の比較グリッドが使っていた MAX_ADDS の値を WORKFLOW.md で確認する
2. 現在の config の MAX_ADDS_BY_PRIORITY と一致しているか確認する
3. 不一致があれば config を修正してから実行する
4. グリッドスクリプト内でも対象外 priority の adds を明示的に固定する
